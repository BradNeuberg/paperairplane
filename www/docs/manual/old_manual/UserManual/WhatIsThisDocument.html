What is this Document?

This document serves several different functions. First, it provides an Introduction and a Walkthrough for new user's to get up to speed on what Paper Airplane and the Two Way Web are all about. Second, it acts as the complete help file for using and interacting with the Paper Airplane application. Third, it is the functional requirements document describing exactly what Paper Airplane 1.0 and the Two Way Web will consist of and what it will not consist of. Finally, this document provides the user-interface description, screenshots, and brand identity needed to create and market the Paper Airplane application.

Please note that neither Paper Airplane nor The Two Way Web exist yet; the screenshots in this document are only mockups.

If you are looking for the following information, you will need to consult other documents than this one:

  • Paper Airplane Security Analysis - This document analyzes various security issues, such as possible security holes inherited with the open-source codebases of Mozilla, gobeProductive, JXTA, etc., the possible motives of attackers to understand what portions of the architecture will be most attractive for hacking, portions of the architecture susceptible to Denial of Service (DoS) attacks, and more.
  • Paper Airplane Architecture - This document provides the blueprints for how the Paper Airplane client application works. It does not include how the peer-to-peer distributed architecture of the Two Way Web will operate; instead, it describes generic interfaces behind which the actual peer to peer code will operate. This document instead discusses the components that are needed for the Paper Airplane client portion and their organization, interfaces to these components, how user-interface code will be arranged, etc.
  • Two Way Web Architecture - This document describes the distributed peer to peer architecture of the Two Way Web, including distributed naming, storage, and PKI, how the code that implements this architecture is implemented on the client, and the API that is exposed to the Paper Airplane browser.
