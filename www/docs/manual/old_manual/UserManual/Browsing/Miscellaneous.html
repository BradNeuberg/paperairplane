Browsing - Miscellaneous

Co-Existence of Mozilla, gobeProductive, JXTA, and Paper Airplane

Paper Airplane is built using open-source components of Mozilla, gobeProductive, and JXTA. It is important that applications written using these pieces, and the original pieces themselves, be able to function on the same machine. For example, both Mozilla and Paper Airplane should be able to be on the same machine. For this to happen Paper Airplane must have different registry values, must store its application values in a folder named Paper Airplane in the user's directory, etc. Both open-source and non-open-source versions of gobeProductive should also be able to be on the same machine. The version of gobeProductive integrated into Paper Airplane should have different registry values, associate itself to open only for .page files, and the File/Save As... dialog window should have its native file-type be Two Way Web Pages (.page) when saving and loading pages. Applications written on top of JXTA should also be able to run at the same time as Paper Airplane without knowing about the existence of Paper Airplane.
