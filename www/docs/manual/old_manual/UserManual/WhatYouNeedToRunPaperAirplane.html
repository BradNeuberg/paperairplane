What You Need to Run Paper Airplane

The following are the baseline requirements for clients to run the Paper Airplane application:

  • A broadband Internet connection, defined as having at least 150K connections upstream and downstream. This connection does not have to be always on, can be behind a firewall, and can be NATed (Network Address Translation).
    • Why: A distributed application like Paper Airplane inherently requires broadband; the numbers indicate that it is simply not feasible to build the Two Way Web above a network that consists primarily of dial-up users.
  • 1024x768 resolution
    • Why: The number of UI elements can not be fit onto the screen at 800x600 resolution
  • Memory requirements not yet set. Probably slightly higher than the Mozilla requirements.
  • Clockspeed requirements not yet set
  • Windows (specific versions not yet set)
    • Why: While Paper Airplane is built on cross-platform technologies (Mozilla, XUL, JXTA, etc.), it is simply not feasible for the small number of current developers to build, test, and deploy Paper Airplane across multiple-platforms at this time.
  • Hard-drive requirements not yet set (will probably be pretty high)
    • Why: While Paper Airplane is free as in money, it is not free as in resources; your payment for using and creating within the Two Way Web are bandwidth and hard-drive resources. The hard-drive requirements might be quite high and will be set based on architectural network requirements.
